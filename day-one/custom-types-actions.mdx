## Overview 

Amplication's **Custom Types & Actions** feature lets you define your API structures and endpoints through a visual interface. This automatically generates production-ready code that follows your organization's standards.

<Note>
The Custom Types & Actions feature is available when using **.NET** or **Node.js** Blueprints.
</Note>

## Core Concepts

### Types

In Amplication, Types primarily refer to Data Transfer Objects (DTOs). These are data structures that define how information moves through your APIs.

Types come in two categories:
- **DTOs:** Custom data structures that group related fields together
- **Primitives:** Basic data types like `String`, `Integer`, `Boolean`, `Date`, and `JSON`

### Actions

Actions define the endpoints of your API. They determine what operations your service can perform.

There are two kinds of Actions:
1. **Default CRUD:** Automatically created Create, Read, Update, and Delete operations
2. **Custom Actions:** Your own endpoints for specific business logic beyond CRUD

<Tip>Think of Actions as verbs and DTOs as nouns - Actions *do* things with your *data* (DTOs).</Tip>

## Working with Modules

Modules help organize your DTOs and Actions into logical groups. While each database entity gets a default Module, you can create custom Modules for better organization.

### Module Management

<AccordionGroup>
    <Accordion title="Create New Module" icon="plus">
        <Steps>
            <Step title="Navigate to Modules">
                In the APIs sidebar, select **"All Modules"**
            </Step>
            <Step title="Start Creation">
                Click **"Add Module"**
            </Step>
            <Step title="Set Module Name">
                Enter a clear, descriptive name
            </Step>
            <Step title="Complete Setup">
                Click **"Create Module"**
            </Step>
        </Steps>

        <Frame caption="Module creation interface">
            <img src="/images/day-one/create-module.png" />
        </Frame>
    </Accordion>

    <Accordion title="Edit Existing Module" icon="pen">
        <Steps>
            <Step title="Select Module">
                Click the Module's name in the APIs tab
            </Step>
            <Step title="Access Edit Mode">
                Click **"Edit Module"**
            </Step>
            <Step title="Update Details">
                Modify the name or description as needed
            </Step>
        </Steps>

        <Frame caption="Module editing screen">
            <img src="/images/day-one/edit-module.png" />
        </Frame>
    </Accordion>

    <Accordion title="Remove Module" icon="trash">
        <Steps>
            <Step title="Choose Module">
                Select the Module to remove
            </Step>
            <Step title="Enter Edit Mode">
                Click **"Edit Module"**
            </Step>
            <Step title="Delete">
                Click the delete icon and confirm
            </Step>
        </Steps>

        <Frame caption="Module deletion interface">
            <img src="/images/day-one/delete-module.png" />
        </Frame>

        <Warning>
            Only Custom Modules can be deleted. Entity-based Modules are protected.
        </Warning>
    </Accordion>
</AccordionGroup>

## Managing DTOs and Enums

DTOs and Enums are the building blocks of your API's type system. They ensure consistent data handling throughout your service.

### Creating Data Structures

<AccordionGroup>
    <Accordion title="Add New DTO" icon="square-plus">
        <Steps>
            <Step title="Begin Creation">
                Click **"Add DTO"** and select **"Add DTO"**
            </Step>

            <Step title="Define Basic Info">
                Input the DTO's name and description

                <Frame caption="DTO creation form">
                    <img src="/images/day-one/create-dto-name-and-description.png" />
                </Frame>
            </Step>

            <Step title="Set Up Properties">
                Click **"Add property"** for each needed field
            </Step>

            <Step title="Configure Each Property">
                For each field, specify:
                * Name
                * Type (Primitive/DTO/Enum)
                * Array status
                * Optional status

                <Frame caption="Property configuration">
                    <img src="/images/day-one/create-dto-add-properties.png" />
                </Frame>
            </Step>
        </Steps>
    </Accordion>

    <Accordion title="Create New Enum" icon="list">
        <Steps>
            <Step title="Start Enum">
                Select **"Add Enum"** from the DTO menu
            </Step>
            <Step title="Set Basic Details">
                Add name and description
            </Step>
            <Step title="Define Members">
                Click **"Add member"** for each enum value
            </Step>
            <Step title="Set Up Members">
                For each one:
                * Define name
                * Set value
            </Step>
        </Steps>
    </Accordion>

    <Accordion title="Modify Existing Types" icon="pen-to-square">
        To update a DTO or Enum:
        1. Find it in its Module
        2. Click its name
        3. Make your changes
    </Accordion>
</AccordionGroup>

## Setting Up Actions

Actions create the contract between your service and its users. Each one maps to an endpoint or operation, with clear input and output types.

### Action Configuration

<AccordionGroup>
    <Accordion title="Create Action" icon="plus">
        <Steps>
            <Step title="Begin Creation">
                Click **"Add Action"**
            </Step>

            <Step title="Set Basic Details">
                Add display name and description

                <Frame caption="Action setup screen">
                    <img src="/images/day-one/create-action-name-and-description.png" />
                </Frame>
            </Step>

            <Step title="Define Types">
                Choose:
                * Input Type
                * Output Type

                <Frame caption="Type configuration">
                    <img src="/images/day-one/create-action-types.png" />
                </Frame>
            </Step>

            <Step title="Configure API Settings">
                For REST APIs:
                * Set HTTP method
                * Define route
                * Choose input source

                <Frame caption="REST configuration">
                    <img src="/images/day-one/create-action-rest-api-settings.png" />
                </Frame>

                For GraphQL:
                * Select operation type

                <Frame caption="GraphQL setup">
                    <img src="/images/day-one/create-action-graphql-settings.png" />
                </Frame>
            </Step>
        </Steps>
    </Accordion>

    <Accordion title="Manage Action Visibility" icon="toggle-on">
        Use the toggle switches to enable or disable specific Actions.

        <Warning>
            Disabled Actions won't generate endpoint code.
        </Warning>

        <Frame caption="Action visibility controls">
            <img src="/images/day-one/all-actions.png" />
        </Frame>
    </Accordion>

    <Accordion title="Update or Remove Actions" icon="pen-to-square">

        * To delete: Use the delete button (custom Actions only)

            Default CRUD Actions cannot be deleted.
        </Note>
    </Accordion>
</AccordionGroup>
