---
title: Create Entity Service Base
description: Creates the base service of an entity
---

## Event Name

`CreateEntityServiceBase`

## Event Parameters

<ParamField body="entityName" type="string">
  The name of the entity for which we are generating the service.
</ParamField>
<ParamField body="entity" type="Entity">
  The entity which we are generating in the service.
</ParamField>
<ParamField body="templateMapping" type="{ [key: string]: any }">
  An object with values that are available in the interpolation process of the template. The default object with the available parameters can be found [here](https://github.com/amplication/amplication/tree/master/packages/data-service-generator/src/server/resource/service/service.base.template.ts). You can manipulate the object by adding new values, or replacing existing values that will be used in the template when creating the service file.
</ParamField>
<ParamField body="passwordFields" type="EntityField[]">
  List of fields of type Password that are used in this entity.
</ParamField>
<ParamField body="serviceId" type="namedTypes.Identifier">
  The Identifier of the service that is used as the class name. e.g. CustomerService.
</ParamField>
<ParamField body="serviceBaseId" type="namedTypes.Identifier">
  The Identifier of the base service that is used as the class name. e.g. CustomerServiceBase.
</ParamField>
<ParamField body="delegateId" type="namedTypes.Identifier">
  The identifier of the Prisma class used to access the entity data.
</ParamField>
<ParamField body="template" type="namedTypes.File">
  A template file that is used to generate the service. This is the default template that is used for this event: [service.base.template.ts](https://github.com/amplication/amplication/tree/master/packages/data-service-generator/src/server/resource/service/service.base.template.ts). You can manipulate the template or replace it completely with a new template in your plugin.
</ParamField>
